  - сохранил реальные данные в приватной папке;
  - добавил приватную папку в `.gitignore`;
  - создал публичный CSV с искусственными данными;
  - прочитал CSV через `open()` и `with`;
  - разделил текст на строки через `splitlines()`;
  - пропустил заголовок с помощью среза `[1:]`;
  - обработал строки циклом `for`;
  - преобразовал текстовый вес в `float`;
  - собрал веса в список через `append()`;
  - рассчитал изменение веса и округлил результат;
  - добавил автоматический выбор между private и sample;
  - сделал путь независимым от директории терминала.

- Что понял своими словами:

  - CSV отличается от Excel тем, что CSV является простым текстовым файлом с одной таблицей и не хранит листы, формулы и оформление;
  - tidy data удобна тем, что каждый столбец хранит один признак, а каждая строка — одно наблюдение;
  - цикл `for` нужен, чтобы последовательно выполнить обработку для каждого элемента списка;
  - `append()` нужен, чтобы добавить новый элемент в конец списка;
  - строка `lines[1:]` нужна, чтобы взять элементы с индекса `1` до конца и пропустить строку заголовков;
  - `float()` используется, потому что CSV читается как текст, а для вычислений вес необходимо преобразовать в число;
  - приватные и публичные данные разделяются, чтобы не публиковать личные данные, но сохранить возможность запустить проект на искусственном примере;
  - путь строится от `__file__`, чтобы находить данные относительно папки скрипта независимо от текущей директории терминала.

- Где застрял:

  - сначала создал папку `sample` внутри `private`;
  - не сразу учёл строку заголовков при подсчёте строк;
  - сначала воспринимал последнюю CSV-строку как одно значение веса;
  - увидел погрешность `2.700000000000003` при вычислениях с `float`.

- Что помогло:

  - последовательная цепочка `read → splitlines → split → float`;
  - проверка типов через `type()`;
  - прогноз результата перед запуском;
  - проверка `.gitignore` через Git;
  - запуск скрипта из двух разных директорий.
