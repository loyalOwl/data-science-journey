# Codex: первое знакомство

## Что это такое

Codex — не только чат с ответами о коде. Это агент, который может читать файлы проекта, искать связи, менять код, запускать команды и тесты, а затем показывать проверенный результат. Важно: Codex ускоряет твоё мышление, но не должен заменять его.

Для обучения используй цикл:

```text
цель → твоя попытка → проверка → объяснение ошибки → исправление → вывод в журнал
```

## Как ставить задачу

Хороший запрос содержит четыре части:

1. Контекст: что ты сейчас изучаешь.
2. Цель: какой результат нужен.
3. Ограничение: что агенту нельзя делать за тебя.
4. Проверка: как понять, что задача выполнена.

Пример:

> Я изучаю функции Python. Посмотри `progress_tracker.py`, но не пиши решение за меня. Объясни первую ошибку и дай одну подсказку. После моей правки запусти тесты.

## Что Codex полезно поручать уже сейчас

- объяснить незнакомый код построчно;
- найти причину ошибки и воспроизвести её;
- придумать упражнения под конкретную тему;
- проверить решение тестами;
- провести код-ревью понятным языком;
- помочь оформить README и проект для портфолио;
- анализировать CSV и строить воспроизводимые отчёты;
- вести план, но сверять его с фактическими результатами.

## Уровни постоянных инструкций

- Сообщение в текущем чате — разовое правило для этой задачи.
- `AGENTS.md` — постоянные правила внутри проекта.
- Skill — повторяемая специализированная процедура.
- Plugin — устанавливаемый набор навыков, инструментов и интеграций.
- Connector/MCP — доступ к внешнему сервису или данным.
- Automation — задача по расписанию или повторяющийся мониторинг.

На старте достаточно чата, `AGENTS.md`, файлов проекта и тестов. Агентные фреймворки пока не нужны: сначала нужно уверенно писать обычный Python.

## Что такое harness engineering

Простыми словами, это создание «рабочего контура», в котором агенту трудно незаметно ошибиться. В контур входят:

- чёткая задача и ограничения;
- структура проекта;
- примеры ожидаемого поведения;
- тесты и команды проверки;
- журнал решений;
- правила безопасности;
- понятный критерий готовности.

В первом проекте таким контуром служат `README`, заготовка функций и автоматические тесты. Codex может помочь, но тесты независимо показывают, работает ли программа.

## Правила безопасного обучения

- Не запускай команду, смысл которой не понимаешь; сначала попроси объяснить.
- Не публикуй пароли, токены, паспортные данные и приватные выгрузки.
- Перед крупным изменением проси показать план и список затрагиваемых файлов.
- Проси запускать тесты, а не принимать фразу «должно работать».
- После готового решения объясни код своими словами — это проверка реального понимания.
