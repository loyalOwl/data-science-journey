# Кирилл 2.0 — путь к первому офферу в Data

Это рабочее пространство для обучения, портфолио и подготовки к поиску работы. Главный принцип: каждая тема заканчивается проверяемым результатом — программой, анализом, графиком, README или опубликованным проектом.

## Главная цель

Получить первый оффер на позицию стажёра или junior-специалиста в аналитике данных / Data Science.

Практичная траектория: сначала стать сильным кандидатом на Data Analyst / Product Analyst internship, параллельно наращивая математику и машинное обучение для перехода в Data Science. Это не отказ от DS, а более широкий вход на рынок.

## С чего начать

1. Прочитать [docs/CODEX_START.md](docs/CODEX_START.md).
2. Настроить терминал по [docs/ENVIRONMENT.md](docs/ENVIRONMENT.md).
3. Открыть [ROADMAP.md](ROADMAP.md) и сверить темп с реальной занятостью.
4. Выполнить первое задание в [projects/01_python_basics/README.md](projects/01_python_basics/README.md).
5. После занятия сделать короткую запись в [LEARNING_LOG.md](LEARNING_LOG.md).

## Структура

```text
.
├── AGENTS.md                 постоянные правила для Codex
├── ROADMAP.md                маршрут до портфолио и собеседований
├── LEARNING_LOG.md           журнал занятий и препятствий
├── docs/
│   ├── CODEX_START.md        краткое введение в Codex
│   └── ENVIRONMENT.md        команды рабочей среды
└── projects/
    └── 01_python_basics/     первый проект и автоматические проверки
```

## Еженедельный ритм

- 5 занятий по 60–90 минут;
- 3 занятия — код и задачи;
- 1 занятие — математика/статистика;
- 1 занятие — проект, README и разбор ошибок;
- в конце недели — короткий отчёт: что сделал, что понял, где застрял.

Количество часов можно менять. Нельзя менять главное: регулярность и наличие результата недели.
