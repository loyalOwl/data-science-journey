# Дорожный видеоплейлист по инженерии AI-агентов

Этот список подготовлен для просмотра с телефона в поездке. Видео расположены не по популярности, а в порядке, в котором новые идеи опираются на предыдущие.

Общая продолжительность основного и расширенного маршрута — примерно **10 часов**. Не нужно запоминать код с экрана. Цель просмотра — понять общую архитектуру и начать узнавать термины. Код мы напишем вместе после поездки.

## Как смотреть

После каждого видео ответь себе на три вопроса:

1. какую проблему решает технология;
2. где она находится внутри AI-системы;
3. что в ней должен контролировать инженер.

Если ответить пока не получается — просто запиши термин. Мы разберём его на практике.

## Маршрут A. Главное об агентах — 2 часа 40 минут

Если времени хватит только на один блок, смотри этот.

1. [Вся база для работы с ИИ: агенты, LLM, RAG и токены](https://www.youtube.com/watch?v=YoHuVfr_rjk) — около 12 минут.
   Общая карта незнакомых терминов. Это вводный обзор, а не источник точных технических деталей.

2. [Контекстное окно LLM](https://www.youtube.com/watch?v=kTO-fN18V_0) — около 9 минут.
   Объясняет, сколько информации одновременно видит модель и почему контекст нужно отбирать.

3. [Function Calling: как модель вызывает твой код](https://www.youtube.com/watch?v=l5o9_8JmTCM) — около 23 минут.
   Показывает переход от чат-бота к системе, которая может выбирать инструменты и выполнять действия.

4. [LLM и Structured Output](https://www.youtube.com/watch?v=kvJkinfe0i8) — около 27 минут.
   Объясняет, зачем получать от модели строгие данные, которые программа способна проверить.

5. [RAG — понятное объяснение](https://www.youtube.com/watch?v=22tkx79icy4) — около 9 минут.
   Знакомит с добавлением внешних знаний в контекст модели.

6. [Векторные базы данных — понятное объяснение](https://www.youtube.com/watch?v=H1EyiGPZwwc) — около 13 минут.
   Объясняет поиск информации по смысловой близости.

7. [Архитектура AI-агентов: LangChain, LangGraph, LlamaIndex и CrewAI](https://www.youtube.com/watch?v=-3RVMVXfkTw) — около 53 минут.
   Сравнивает несколько способов собирать агентные системы. Важно понять различия, а не выбрать любимый фреймворк.

8. [Что такое MCP](https://www.youtube.com/watch?v=yMkdvg_U_Nc) — около 16 минут.
   Показывает стандартный способ подключения внешних инструментов и данных к AI-приложениям.

После маршрута A нужно понимать общую цепочку:

```text
задача → контекст → LLM → структурированное решение → tool → результат → следующий шаг
```

## Маршрут B. Инженерная система вокруг агента — ещё 4 часа 15 минут

Этот блок показывает, как превратить идею агента в работающий сервис.

9. [REST API, HTTP, клиент-сервер, Swagger и OpenAPI](https://www.youtube.com/watch?v=mkpJIZWQlHY) — около 28 минут.
   Объясняет, как CRM, сайт или другой продукт будет обращаться к нашему агенту.

10. [FastAPI простым языком](https://www.youtube.com/watch?v=JqfbfraMjW8) — около 12 минут.
    Показывает инструмент, с помощью которого мы сделаем API для запуска агентов.

11. [Асинхронность в Python без воды](https://www.youtube.com/watch?v=cDXzxd9Nf7o) — около 21 минуты.
    Объясняет, как сервер обслуживает другую работу, пока ожидает модель, базу данных или внешний API.

12. [Вся база SQL для начинающих](https://www.youtube.com/watch?v=5sG9kmXYsKU) — около 1 часа 19 минут.
    Даёт основу хранения клиентов, задач, запусков агента, статусов и результатов. Можно смотреть частями.

13. [Docker за 20 минут](https://www.youtube.com/watch?v=Z_cUS7kCAsE) — около 21 минуты.
    Объясняет упаковку программы вместе с её окружением для воспроизводимого запуска.

14. [Введение в AI Agents на базе LangGraph](https://www.youtube.com/watch?v=mqV9COmt6zk) — около 1 часа 35 минут.
    Показывает состояния и переходы долгоживущего агента. Не заучивай синтаксис: ищи знакомые элементы agent loop.

После маршрута B нужно видеть, что production-агент — это не только модель:

```text
API + бизнес-логика + база данных + agent loop + инструменты + контейнер
```

## Маршрут C. Качество и выпуск в production — ещё 2 часа 30 минут

15. [Оценка и тестирование LLM-агентов](https://www.youtube.com/watch?v=cJwoF_k1llI) — около 1 часа 11 минут.
    Объясняет evals: проверку ответа, выбора инструмента и всей траектории агента.

16. [GitHub Actions для CI/CD](https://www.youtube.com/watch?v=zn5T7FkpaTA) — около 35 минут.
    Показывает автоматический запуск тестов перед выпуском новой версии.

17. [Git простым языком](https://www.youtube.com/watch?v=cG-8NnH4x94) — около 44 минут.
    Собирает в единую модель staging, commits и историю. Можно пропустить, если наши практические занятия по Git уже понятны.

После маршрута C нужно понимать принцип:

> Агент становится production-системой, когда его поведение можно воспроизвести, проверить, наблюдать и безопасно обновить.

## Что записывать в заметки

Не делай подробный конспект каждого ролика. Достаточно такого шаблона:

```text
Название темы:
Главная проблема:
Как решается:
Где находится в архитектуре агента:
Что осталось непонятно:
```

Особенно отмечай вопросы про:

- различие workflow и автономного агента;
- structured outputs и function calling;
- state, memory и context;
- RAG и обычную базу данных;
- tool, API и MCP;
- tests и evals;
- agent loop и stopping conditions;
- ответственность модели, программы и человека.

После поездки мы не будем начинать курс заново. Используем твои вопросы из заметок и продолжим текущий проект, постепенно заменяя детерминированные правила LLM-компонентами.
