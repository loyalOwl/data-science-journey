# Кирилл 2.0 — путь к профессии AI Engineer / Agent Engineer

Это рабочее пространство для обучения и подготовки к реальным задачам по разработке AI-агентов. Главный принцип: каждая тема заканчивается проверяемым результатом — программой, API, тестом, eval-набором или развёрнутым сервисом.

## Главная цель

За 1–2 месяца подготовиться к работе AI Engineer / Agent Engineer: научиться строить, оценивать, развёртывать и сопровождать автономных агентов для разных бизнес-процессов.

Основной маршрут:

```text
инженерная база → LLM → backend → RAG → agent architecture → harness → MCP → evals → деплой
```

Подробный план и критерии готовности находятся в [ROADMAP.md](ROADMAP.md).
Целевой профессиональный профиль описан в [docs/AI_ENGINEER_PROFILE.md](docs/AI_ENGINEER_PROFILE.md).
Разбор SoloGPT и «Колли» как компаний-полигонов находится в [docs/PRODUCT_REFERENCE.md](docs/PRODUCT_REFERENCE.md).

## Связь с прежним маршрутом

Переход от Data Science к инженерии AI-агентов не обнуляет предыдущую работу. Python, Git, тесты, CSV, SQL и анализ данных становятся фундаментом для разработки агентов, хранения диалогов и оценки их качества. Классическое машинное обучение и углублённая статистика временно отложены: к ним можно вернуться после адаптации на работе или когда они понадобятся конкретному продукту.

## С чего начать

1. Прочитать [docs/CODEX_START.md](docs/CODEX_START.md).
2. Настроить терминал по [docs/ENVIRONMENT.md](docs/ENVIRONMENT.md).
3. Открыть [ROADMAP.md](ROADMAP.md) и работать по текущему этапу.
4. После нескольких тем или упражнений добавить итог в [LEARNING_LOG.md](LEARNING_LOG.md).

## Ритм работы

- один небольшой проверяемый шаг за раз;
- после изменения — тест или явная ручная проверка;
- после нескольких тем — запись в `LEARNING_LOG.md`;
- после законченного блока — осмысленный commit и push;
- каждую неделю — демонстрация работающего результата.
